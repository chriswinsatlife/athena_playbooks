# Playbook Analysis Prompt

Based on the research findings for the **{{ slug }}** playbook, generate a comprehensive research synthesis.

**CRITICAL OUTPUT INSTRUCTION #1: NO PROCESS WORDS IN OUTPUT.** NEVER use words like 'Verified', 'Sourced', 'Confirmed', 'Researched' in your text.

**CRITICAL OUTPUT INSTRUCTION #2: NO INLINE CITATIONS.** Do **NOT** add `[Source Name, Year]` or any citation within body text. The text should be clean.

**CRITICAL OUTPUT INSTRUCTION #3: MANDATORY SOURCE LIST.** Include a comprehensive source list at the end with clickable URLs.

**CRITICAL OUTPUT INSTRUCTION #4: NO HYPE.** Avoid "revolutionary", "game-changing", "transform your life". Data-driven, specific, actionable.

**Structure:**

1. **Executive Summary**
   - 1-2 paragraphs: what this playbook enables, core value proposition, key workflows

2. **Problem Analysis**
   - Specific pain points this solves
   - Cost of not having this system (time, stress, missed opportunities)

3. **Workflow Specifications**
   - For each workflow: trigger, steps, owner (client vs EA), time estimate, outcome
   - Use bullet points for steps

4. **Tool Stack**
   - Recommended tools with specific use cases
   - Template structures
   - Integration points

5. **Delegation Blueprint**
   - Client responsibilities (should be minimal)
   - EA responsibilities (bulk of work)
   - Handoff patterns
   - Approval flows

6. **Edge Case Handling**
   - Each edge case with specific handling instructions

7. **Success Metrics**
   - Quantitative: specific numbers
   - Qualitative: how it feels

8. **Sources Used**
   - Bulleted list
   - Each source as clickable Markdown link
   - Format: `- [Description](URL)`

**Tone:** Objective, specific, actionable. No hype. Stick to evidence.
